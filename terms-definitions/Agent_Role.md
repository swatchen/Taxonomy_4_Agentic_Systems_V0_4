[[def: Agent Role]]

~The storehouse and management layer for persistent perspectives that can be instantiated as Agents (Live). An Agent Roleâ€™s functions are split between one or more Workflows. An Agent Role is defined as the Sub-Workspace where Workflows relevant to that Role run, and where Workload Execution Records are accumulated. It does not have its own perspective; instead, it is an addressable reservoir of context that can be assembled into temporary perspectives for immediate tasks. The Agent Role provides the illusion of a continuous, persistent Agent to the user, even if the underlying Live Agents are ephemeral.
